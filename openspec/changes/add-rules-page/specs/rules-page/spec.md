# 竞赛规则页面规格说明

## ADDED Requirements

### Requirement: 可配置的竞赛规则内容
系统管理员**SHALL**能在 `simple-config.js` 中配置竞赛规则的内容，包括标题、规则条目和页面设置。

#### Scenario: 管理员配置竞赛规则
- 当管理员需要更新竞赛规则时
- 他们**SHALL**能直接编辑 `simple-config.js` 文件
- 系统**MUST**能够读取并显示更新后的规则内容
- 无需修改代码即可完成规则更新

### Requirement: 导航栏访问
用户**SHALL**能通过导航栏中的"竞赛规则"按钮直接访问竞赛规则页面。

#### Scenario: 用户点击导航栏按钮
- 当用户在任何页面时
- 点击导航栏中的"竞赛规则"按钮
- 系统**MUST**显示竞赛规则页面
- 当前页面**SHALL**被隐藏

### Requirement: 首页跳转
用户在首页点击任意位置后，系统**SHALL**跳转到竞赛规则页面而非选题页面。

#### Scenario: 用户开始使用系统
- 当用户在首页时
- 点击页面任意位置
- 系统**MUST**显示竞赛规则页面
- 为用户提供了解规则的机会

### Requirement: 一致的页面样式
竞赛规则页面**SHALL**与现有系统的页面样式保持一致，包括颜色、字体和布局。

#### Scenario: 用户浏览竞赛规则
- 当用户查看竞赛规则页面时
- 页面**MUST**使用与其他页面相同的主题颜色
- 字体大小和样式**SHALL**保持一致
- 响应式设计**MUST**正常工作

### Requirement: 页面导航集成
竞赛规则页面**SHALL**完全集成到现有的页面导航系统中。

#### Scenario: 用户在不同页面间切换
- 当用户从竞赛规则页面切换到其他页面时
- **SHALL**使用与现有页面相同的切换机制
- 页面状态**MUST**正确管理
- 导航栏按钮状态**MUST**正确更新

## MODIFIED Requirements

### Requirement: 首页点击逻辑
修改首页的点击事件处理逻辑，**SHALL**将跳转目标从选题页改为竞赛规则页。

#### Scenario: 用户从首页开始
- 当用户首次访问系统时
- 在首页点击任意位置
- 系统**MUST**显示竞赛规则页面
- 而不是直接跳转到选题页面

## REMOVED Requirements

### Requirement: 无
此功能添加不会移除任何现有功能。
